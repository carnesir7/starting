    Color Usage Guidelines

  • Do's

    • Use primary colors for main actions and brand elements
    • Use semantic colors consistently across components
    • Ensure sufficient contrast for accessibility
    • Use muted colors for less important elements
  • Don'ts

    • Don't use destructive colors for non-error states
    • Don't mix custom colors with system colors
    • Don't use pure black or white without considering contrast
    • Don't override color tokens without good reason

Buttons

Button Variants

Primary Button

Used for the main action in a section. Bold, filled with primary color.

Secondary Button

Used for alternative actions. Outlined with subtle background.

Used for tertiary actions. Outlined with transparent background.

Additional Variants

Destructive Button

Used for dangerous actions like delete.

Link Button

Styled like a text link.

Button Sizes

Small: For compact interfaces

Default: Standard size for most use cases

Large: For emphasis and mobile interfaces

Button Usage Guidelines

Best Practices

  • Use only one primary button per section
  • Use descriptive, action-oriented labels
  • Maintain consistent spacing between buttons
  • Use appropriate sizes for the context
  • Consider touch targets for mobile devices
